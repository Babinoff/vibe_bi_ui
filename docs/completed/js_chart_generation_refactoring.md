# План рефакторинга: Переход на генерацию JS-кода для конфигурации графиков (Visualization Node)

## 1. Совместимость с библиотеками (Ответ на вопрос)
Да, генерация JavaScript-кода вместо статического JSON **будет отлично работать со всеми тремя библиотеками** (ECharts, Chart.js, Plotly). 

**Почему это работает лучше:**
Вместо того чтобы пытаться использовать специфичные и ограниченные декларативные методы библиотек (вроде `dataset.transform` в ECharts), LLM сможет использовать стандартные методы JavaScript (`Array.prototype.reduce`, `map`, `filter`, `Set`). Модель будет получать "плоские" данные (long-format) и сама программно формировать из них:
- **Для Chart.js**: массив `datasets`, где каждый объект — это отдельная категория со своим цветом.
- **Для Plotly**: массив `traces`, где каждый объект — это отдельная линия/столбец для категории.
- **Для ECharts**: классический `series` массив без необходимости сложных трансформаций внутри самого ECharts.

Таким образом, JS становится универсальным "адаптером" данных перед передачей в любую библиотеку.
